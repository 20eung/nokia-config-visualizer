# Release Notes - v5.6.3

> **Date**: 2026-03-31
> **Branch**: `main`
> **Type**: Bug Fix (Patch Release)

## Summary

Grafana Diagram 패널에서 Mermaid 노드 라벨의 HTML 속성 이중 따옴표 충돌 및 노드 ID 하이픈 파싱 오류 수정.

## Bug Fixes

### 1. Mermaid 노드 라벨 HTML 속성 따옴표 충돌

**증상**: Grafana Diagram 패널에서 SAP/SERVICE 노드가 빈 박스로 렌더링되거나 레이아웃이 깨짐

**원인**: Mermaid 노드 라벨은 이중 따옴표로 감싸는 `["..."]` 형식인데, 내부 HTML 속성(`style="..."`, `class="..."`)도 이중 따옴표를 사용하여 파싱 충돌 발생.

```
// Before (충돌)
SAP_node["<div style="text-align: left">...</div>"]
//              ^^^^ 파싱 여기서 라벨이 끊김

// After (수정)
SAP_node["<div style='text-align: left'>...</div>"]
```

**수정 범위**:
- `qosHighlight()` 함수: `<span class="qos-hl" style="...">` → 단일 따옴표
- Epipe/VPLS/VPRN/BGP/OSPF/STATIC 노드 라벨의 `<div style="text-align: left">` → 단일 따옴표 (총 9곳)

### 2. 서브그래프/노드 ID 하이픈 파싱 오류

**증상**: `SK-Net_DDC2F_7750SR-7_MPLS_1` 처럼 하이픈이 포함된 hostname의 다이어그램만 깨지는 현상

**원인**: Mermaid가 서브그래프/노드 ID의 하이픈(`-`)을 화살표 문법(`-->`)으로 오파싱. `sanitizeNodeId()`가 하이픈을 허용 문자로 유지하고 있었음.

```typescript
// Before
return id.replace(/[^a-zA-Z0-9_-]/g, '_');  // 하이픈 허용

// After
return id.replace(/[^a-zA-Z0-9_]/g, '_');   // 하이픈도 언더스코어로 변환
```

**효과**: `HOST_SK-Net_DDC2F_7750SR-7_MPLS_1_G0_0` → `HOST_SK_Net_DDC2F_7750SR_7_MPLS_1_G0_0`

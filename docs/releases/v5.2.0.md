# Release Notes - v5.2.0

> **Date**: 2026-03-02
> **Branch**: `main`
> **Type**: Feature (Minor Release)

## Summary

Epipe 현행화 상태 표시, VPRN/IES HA 뱃지, VPLS (L2) SAP 필터링 추가. 다이어그램 가독성과 서비스 상태 가시성을 대폭 향상했습니다.

## New Features

### Epipe 현행화 상태 표시

- Epipe는 반드시 2개 장비에 설정되어야 정상. 1개 장비만 설정된 경우 "현행화 필요" 표시
- Epipe 그룹 헤더에 `✅ N` (정상), `⚠️ N` (비정상) 카운트 뱃지 표시
- 비정상 Epipe 카드에 오렌지 왼쪽 보더 + `⚠️ 현행화 필요 (N개 장비)` 뱃지

### VPRN/IES HA 뱃지

- VPRN 서비스: 정적 라우트 분석으로 HA 감지 → 서비스 헤더에 `HA` 뱃지 표시
- IES 인터페이스: 동일 prefix에 2개 next-hop 존재 시 인터페이스별 `HA` 뱃지 표시

### VPLS (L2) SAP 필터링

- Nokia 명명 규칙 `To_<hostname>(L2)` 패턴의 SAP는 장비 간 백홀 연결이므로 구성도에서 제외
- SAP `description` 및 포트 `portDescription` 양쪽 모두 검사하여 `(L2)` 종료 시 필터링
- 고객 facing SAP만 표시되어 구성도 가독성 향상

### 대시보드 레이아웃 개선 (v5.1 후속)

- 통계 카드 항상 4열 1줄, 브라우저 스크롤 기반으로 전환
- 헤더 버튼 좁은 화면에서 항상 표시
- 앱 제목 반응형: 640px 이상 풀네임 표시

## Bug Fixes

- 이중화 필터 자동 선택 오류: `services` 대신 `filteredServices` 기반으로 자동 선택
- 전체 선택 중복 키 오류: `Array.from(new Set(allKeys))`로 중복 제거
- TypeScript 빌드 오류: `filterType === 'ha' ? 'all' : filterType`으로 처리
- `/api/config` Rate Limit 제외 수정

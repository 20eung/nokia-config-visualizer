# Release Notes - v4.7.0

> **Date**: 2026-02-20
> **Branch**: `main`
> **Type**: Feature (Minor Release)

## Summary

자동 Config 로딩 기능 추가. `./configs` 폴더에 파일을 복사하면 자동으로 감지하여 프론트엔드에 실시간 반영합니다. Docker volume mount로 백업 폴더와 연결하여 무인 운영이 가능합니다.

## New Features

### 자동 Config 로딩 (Auto Config Loading)

- `./configs` 폴더 자동 감시 (chokidar v4.0.1)
- 파일 추가/변경/삭제 시 실시간 자동 감지
- WebSocket으로 프론트엔드에 즉시 알림 (ws v8.18.0)
- hostname별 최신 파일만 자동 필터링
- 다양한 파일명 형식 지원:
  - `router1-20260219.txt` (하이픈 + YYYYMMDD)
  - `router1_20260219.txt` (언더스코어 + YYYYMMDD)
  - `router1-2026-02-19.txt` (하이픈 + YYYY-MM-DD)

### 멀티파일 선택

- 모든 파일 기본 활성화
- 클릭으로 개별 파일 토글 (추가/제거)

### 접을 수 있는 사이드바

- Config 파일 목록을 접어 화면 공간 확보 (기본: 접힘)
- 0.3s 부드러운 전환 애니메이션

## Backend Changes

**새 파일**:
- `server/src/utils/configFilenameParser.ts` — 파일명 파싱 유틸리티
- `server/src/services/fileWatcher.ts` — File Watcher 서비스 (chokidar)
- `server/src/services/websocket.ts` — WebSocket 서버 (ws)
- `server/src/routes/config.ts` — Config 파일 관리 API

**새 API 엔드포인트**:
- `GET /api/config/files` — 파일 목록 조회
- `GET /api/config/file/:filename` — 파일 다운로드
- `GET /api/config/watch-status` — 감시 상태 조회
- `GET /api/config/groups` — hostname별 파일 그룹

## Frontend Changes

**새 파일**:
- `src/hooks/useConfigWebSocket.ts` — WebSocket 연결 관리 Custom Hook
- `src/components/v3/ConfigFileList.tsx` — Config 파일 목록 컴포넌트
- `src/components/v3/FolderPathSettings.tsx` — 폴더 경로 안내 컴포넌트

## Bug Fixes

- 포트 매핑 오류: `3001:3001` → `3001:3000`
- Rate Limit 429 오류: 85개 파일 동시 로딩 시 발생하던 문제 해결 (`/api/config` Rate Limit 제외)
- 단일 파일만 로딩되던 문제: `activeFile` → `activeFiles` 배열로 변경

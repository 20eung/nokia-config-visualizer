# Release Notes - v4.4.0

> **Type**: Feature + Bug Fix
> **Full Changelog**: https://github.com/20eung/nokia-config-visualizer/compare/v4.3.0...v4.4.0

## Summary

Dictionary 구조를 3-Field로 개선하여 Config 검색 키워드와 사용자 검색어를 명확히 분리하고 양방향 검색 지원.

## New Features

### 3-Field Dictionary Structure

- **역할 분리**:
  - `name`: 그룹 대표 이름 (예: "SK쉴더스")
  - `configKeywords`: Config description에 실제 존재하는 키워드 (예: ["Bizen", "ADTCAPS"])
  - `searchAliases`: 사용자 검색어 (예: ["SK쉴더스", "ISAC", "인포섹"])
- **양방향 검색**: configKeywords와 searchAliases 모두 검색 가능
- **다중 키워드 지원**: 여러 Config 키워드를 하나의 entry로 그룹화

### Dictionary Editor UI 개선

- **3컬럼 테이블**: 그룹명(15%), Config 키워드(35%), 검색 별칭(45%)
- 3개 컬럼 모두 클릭 정렬 지원

## Bug Fixes

- Docker 빌드 오류: DictionaryEditor.tsx TypeScript 컴파일 에러 27개 수정
- 타입 불일치: `dictionaryApi.ts`의 DictionaryGenerateResponse 타입 서버와 동기화

## Migration Guide (v4.3.0 → v4.4.0)

기존 v4.3.0 사전은 AI 재생성 권장:
- Dictionary Editor에서 "전체 삭제" 후 "AI 자동 생성" 클릭
- AI가 자동으로 configKeywords와 searchAliases를 분류

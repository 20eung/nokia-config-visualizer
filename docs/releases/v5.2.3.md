## Bug Fixes

### Services 헤더 서비스 수 이중집계 수정

**증상**: Services 헤더에 표시되는 서비스 수가 대시보드 통계 카드의 2배

**원인**: Epipe는 2개 장비에 동일 서비스 ID가 존재 → `filteredServices.reduce()` 집계 시 2배 카운트

**수정**: `filteredServices.reduce()` → 이미 고유 서비스 ID로 그룹화된 변수 합산으로 변경
```
epipeServices.length + vplsServices.length + vprnServices.length + iesInterfaceCount
```

Services 헤더 수치(616)가 대시보드 통계 카드와 일치.

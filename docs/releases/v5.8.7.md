# Release Notes - v5.8.7

> **Date**: 2026-04-22
> **Branch**: `main`
> **Type**: Infrastructure / Security

## Overview

Nokia Config Visualizer를 포털 전용 접속으로 전환.
Authentik Forward Auth를 NCV 자체 nginx에서 제거하고, 포털 nginx가 인증을 일괄 처리하도록 변경.
`VITE_API_URL` 빌드 인자로 API/WebSocket 경로를 동적으로 주입.

## 변경 내용

### nginx.conf — Authentik 인증 전체 제거

포털 nginx가 `/services/visualizer/` 경유 시 이미 인증을 처리하므로 NCV 자체 인증 불필요.

- `map $http_sec_fetch_dest` 블록 제거
- `/outpost.goauthentik.io` 프록시 location 제거
- `@goauthentik_proxy_signin` named location 제거
- `/api/` location: auth_request 제거, 순수 proxy_pass만 유지
- `/ws` location: auth_request 제거, 순수 WebSocket proxy_pass만 유지

### vite.config.ts — base path 동적화

```typescript
base: process.env.VITE_API_URL ? `${process.env.VITE_API_URL}/` : '/',
```

포털 경유 시 `/services/visualizer/` 기준으로 정적 자산 경로 자동 설정.

### Dockerfile — build arg 지원 추가

```dockerfile
ARG VITE_API_URL
ENV VITE_API_URL=${VITE_API_URL}
RUN npm run build
```

### docker-compose.yml

```yaml
build:
  args:
    - VITE_API_URL=/services/visualizer
environment:
  - CORS_ORIGIN=...,https://portal.hub.sk-net.com
```

### src/utils/apiBase.ts (신규)

```typescript
export const API_BASE = (import.meta.env['VITE_API_URL'] as string | undefined) ?? '';
```

### API/WebSocket URL 통일

모든 fetch 및 WebSocket URL을 `API_BASE` 기준으로 변경:

- `src/hooks/useConfigWebSocket.ts`
- `src/pages/V3Page.tsx`
- `src/services/chatApi.ts`
- `src/services/configApi.ts`
- `src/services/dictionaryApi.ts`

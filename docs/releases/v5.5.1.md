## 🐛 버그 수정

### Copy PNG 클립보드 복사 기능 수정

#### 문제 상황
- **증상**: 다이어그램 패널의 'Copy PNG' 버튼이 클립보드에 이미지를 복사하는 대신 PNG 파일을 다운로드
- **영향 범위**: NPM(nginx proxy manager) 등의 리버스 프록시를 통해 HTTPS로 접속하는 모든 사용자
- **원인**: 브라우저의 Clipboard API가 Permissions Policy 헤더 누락으로 인해 차단됨

#### 해결 방법

##### 1. 애플리케이션 레벨 개선
- **파일**: `src/components/v3/ServiceDiagram.tsx`
- **변경 내용**:
  ```typescript
  // Clipboard API 우선 시도
  if (navigator.clipboard && navigator.clipboard.write) {
      try {
          await navigator.clipboard.write([
              new ClipboardItem({ 'image/png': blob })
          ]);
          console.log('Successfully copied to clipboard');
          setCopied(true);
          setTimeout(() => setCopied(false), 2000);
          return; // 성공 시 종료
      } catch (clipboardError) {
          console.warn('Clipboard API blocked by policy, falling back to download:', clipboardError);
          // Permissions Policy로 차단된 경우 다운로드 fallback으로 진행
      }
  }

  // Fallback: PNG 파일로 다운로드
  console.log('Downloading as PNG file (clipboard blocked by proxy policy)');
  const url = URL.createObjectURL(blob);
  const a = document.createElement('a');
  a.href = url;
  a.download = `${diagramName || 'diagram'}.png`;
  a.click();
  URL.revokeObjectURL(url);
  ```

- **개선 사항**:
  - Clipboard API 우선 시도, 실패 시 자동으로 다운로드 fallback
  - 상세한 콘솔 로그 출력 (디버깅 용이)
  - Permissions Policy 차단 시 사용자 친화적 안내 메시지

##### 2. Docker 컨테이너 nginx 설정
- **파일**: `nginx.conf`
- **변경 내용**: 각 location 블록에 Permissions-Policy 헤더 추가
  ```nginx
  location / {
      try_files $uri $uri/ /index.html;

      # 보안 헤더
      add_header Permissions-Policy "clipboard-write=*, clipboard-read=*" always;
      # ... 기타 보안 헤더
  }

  location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
      expires 1y;
      add_header Cache-Control "public, immutable";

      # 보안 헤더
      add_header Permissions-Policy "clipboard-write=*, clipboard-read=*" always;
      # ... 기타 보안 헤더
  }
  ```

- **nginx 주의사항**:
  - `add_header`는 location 블록에서 사용 시 상위 블록의 헤더를 상속하지 않음
  - 각 location 블록에 명시적으로 추가 필요

##### 3. NPM(nginx proxy manager) 설정
- **위치**: NPM 관리 페이지 → Proxy Hosts → 해당 호스트 → Advanced 탭
- **추가할 설정**:
  ```nginx
  # Clipboard API 허용
  add_header Permissions-Policy "clipboard-write=(self), clipboard-read=(self)" always;
  ```

- **문법 차이**:
  - `clipboard-write=*`: 모든 출처 허용 (일부 브라우저에서 무시됨)
  - `clipboard-write=(self)`: 같은 출처만 허용 (권장, Chrome/Safari 호환)

- **적용 방법**:
  1. NPM 설정 저장
  2. 프록시 호스트 재시작 (Disable → Enable)
  3. 브라우저 하드 리프레시 (Cmd+Shift+R / Ctrl+Shift+R)

#### 검증 방법

##### 1. 헤더 확인
```bash
curl -I https://your-domain.com | grep -i permissions
# 출력: permissions-policy: clipboard-write=(self), clipboard-read=(self)
```

##### 2. 브라우저 콘솔 확인
개발자 도구(F12) → Console 탭에서 'Copy PNG' 클릭 후:
- ✅ 정상: `Blob generated: XXXXX bytes` → `Successfully copied to clipboard`
- ❌ 차단: `[Violation] Permissions policy violation: The Clipboard API has been blocked`

##### 3. 클립보드 테스트
- 이미지 편집기(그림판, Photoshop, PowerPoint 등)에서 붙여넣기(Ctrl+V / Cmd+V)
- PNG 이미지가 붙여넣어지면 성공

## 🔧 기술 세부사항

### Clipboard API Permissions Policy

#### 브라우저 보안 정책
- Clipboard API는 **Secure Context** (HTTPS 또는 localhost)에서만 동작
- Permissions Policy 헤더가 없으면 브라우저가 API 접근 차단
- 크로스 오리진 iframe에서는 명시적 허용 필요

#### Permissions-Policy 문법
```nginx
# 문법 1: 모든 출처 허용 (일부 브라우저 무시)
add_header Permissions-Policy "clipboard-write=*, clipboard-read=*" always;

# 문법 2: 같은 출처만 허용 (권장)
add_header Permissions-Policy "clipboard-write=(self), clipboard-read=(self)" always;

# 문법 3: 특정 도메인 허용
add_header Permissions-Policy "clipboard-write=(self \"https://example.com\"), clipboard-read=(self)" always;
```

#### 브라우저 호환성
| 브라우저 | `*` (모든 출처) | `(self)` (같은 출처) | 지원 버전 |
|---------|-----------------|---------------------|-----------|
| Chrome  | ⚠️ 무시될 수 있음 | ✅ 권장             | 89+       |
| Safari  | ❌ 미지원         | ✅ 권장             | 15.4+     |
| Firefox | ✅ 지원          | ✅ 지원             | 87+       |
| Edge    | ⚠️ 무시될 수 있음 | ✅ 권장             | 89+       |

### nginx add_header 동작 원리
- `add_header`는 **현재 블록에서 사용 시 상위 블록 헤더를 상속하지 않음**
- server 블록에 정의된 헤더는 location 블록에서 `add_header` 사용 시 무시됨
- **해결책**: 각 location 블록에 필요한 헤더를 모두 명시적으로 추가

## 🔄 마이그레이션 가이드

### 기존 사용자 (v5.5.0 이하)

#### Docker 환경
1. 최신 코드 pull: `git pull origin main`
2. 컨테이너 재빌드: `sudo docker-compose build`
3. 컨테이너 재시작: `sudo docker-compose up -d`

#### NPM 프록시 사용자 (추가 작업 필요)
1. NPM 관리 페이지 접속
2. Proxy Hosts → 해당 호스트 선택 → Edit
3. Advanced 탭 클릭
4. Custom Nginx Configuration에 추가:
   ```nginx
   add_header Permissions-Policy "clipboard-write=(self), clipboard-read=(self)" always;
   ```
5. Save 클릭
6. 프록시 재시작 (Disable → Enable)
7. 브라우저에서 Cmd+Shift+R (하드 리프레시)

#### 검증
- 'Copy PNG' 버튼 클릭
- 이미지 편집기에 붙여넣기 테스트
- 브라우저 콘솔에서 에러 확인

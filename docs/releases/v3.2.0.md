# Release Notes - v3.2.0

> **Type**: Feature + Bug Fix
> **Full Changelog**: https://github.com/20eung/nokia-config-visualizer/compare/v3.1.0...v3.2.0

## Summary

QoS 색상 강조, VPRN 라우팅 중간 노드, SAP 파싱 전면 개편.

## New Features

### VPRN Native Routing Middle Nodes

- **BGP/OSPF/STATIC 분리 노드**: 라우팅 정보가 서비스 노드에서 분리되어 독립적인 중간 노드로 표시
  - BGP 노드: Router-ID, Split-Horizon, Group/Peer 정보, Peer IP → 인터페이스 서브넷 자동 매칭
  - OSPF 노드: Area/Interface 정보, Interface Name 직접 비교 매칭
  - STATIC 노드: Next-Hop별 별도 노드, Route 수 카운트, Next-Hop → 인터페이스 서브넷 매칭
- **3단 레이아웃**: Interface → Routing Nodes → Service Node
- V1 어댑터 제거, VPRN을 V3 네이티브 `generateServiceDiagram()`으로 직접 렌더링

### QoS 색상 강조

- Epipe/VPLS/VPRN SAP 노드의 In-QoS/Out-QoS에 녹색 배경(`#4caf50`) + 흰색 글자 적용
- `.qos-hl` CSS 클래스 기반 (Mermaid SVG foreignObject 호환)

### QoS Rate KMG 변환

- `sap-ingress`/`sap-egress` 정책 정의에서 실제 rate 값 추출
- 7210SAS(meter 기반), 7750SR/7450ESS/7705SAR(queue 기반) 모두 지원
- 100,000 kbps → `100M`, 1,000,000 kbps → `1G`, unlimited → `Max`

### Shutdown 필터링

- `adminState='down'` 항목을 다이어그램에서 자동 제외 (파싱 데이터는 보존)

## Bug Fixes

- **SAP 파싱 누락**: regex lookahead 방식 → position-based 방식으로 전면 교체 (마지막 SAP 누락 치명적 버그 수정)
- **SAP adminState 오탐**: `no shutdown`이 `shutdown`으로 잘못 판정되는 문제 수정
- **VLAN-less SAP 파싱**: `sap 4/2/23 create` 형식 파싱 누락 수정
- **멀티 IES 인터페이스 병합**: 동일 hostname의 여러 IES 서비스 인터페이스 미통합 수정
- **VPLS 호스트 정렬**: hostname 기준 오름차순 정렬 적용

## Technical Updates

- 들여쓰기 레벨 DIAGRAM_RULES.md 기준으로 전면 조정
- 라벨 케이싱 통일 (SPEED→Speed, AUTONEGO→AutoNego 등)
- DIAGRAM_RULES.md 대규모 업데이트

# Release Notes - v5.8.1

> **Date**: 2026-04-10
> **Branch**: `main`
> **Type**: Performance

## 개요

v5.8.0 React 성능 최적화 이후 Vercel React Best Practices 재점검으로 발견된 잔존 이슈 4종 해결.

## 변경 사항

### ⚡ 성능 최적화

**`src/pages/V3Page.tsx`**
- `groupedDiagrams` useMemo 추출: JSX 내부에서 직접 실행되던 `diagrams.reduce()` 그루핑을 useMemo로 추출. `diagrams` 배열이 변경될 때만 재계산되므로 서비스 50개+ 환경에서 매 렌더링마다 발생하던 O(n) 비용 제거.

**`src/components/v3/ServiceDiagram.tsx`**
- `handleCopyCode` useCallback 적용: `[diagram]` 의존성. 동일 컴포넌트의 `handleCopyImagePNG`와 일관성 확보.
- `handleZoomIn/handleZoomOut/handleResetZoom` useCallback 적용: `[]` 의존성 (functional updater 패턴으로 안전). 줌 제어 3개 함수 참조 안정화.

**`src/components/v3/Dashboard.tsx`**
- `STAT_CARD_DEFS` 컴포넌트 외부 상수로 분리: 색상/라벨 등 정적 설정을 컴포넌트 외부로 이동, 의도 명확화.
- `statCards` useMemo 제거: `totalStats` 변경 시 리렌더 자체가 발생하므로 useMemo 불필요. 단순 `.map()`으로 교체.

## 영향 범위

- UI 동작 변경 없음 (순수 최적화)
- TypeScript 컴파일 오류 없음
- 빌드 크기 변동 없음

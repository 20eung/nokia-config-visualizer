## Summary

Unicode Non-Breaking Hyphen(U+2011)이 포함된 hostname 검색이 실패하는 버그 수정.

## Bug Fixes

### Unicode 하이픈 문자 검색 오류 수정

**증상**: `SK-Net_Gyowon_7705SAR-8_MPLS` 같은 hostname을 `sk-net` 또는 `gyowon`으로 검색 불가

**원인**: hostname에 U+2011 Non-Breaking Hyphen 사용. JavaScript의 `.toLowerCase().includes()`는 Unicode 정규화 미지원.

**해결**: `normalizeSearchString()` 함수 추가

```typescript
function normalizeSearchString(str: string): string {
  return str
    .normalize('NFKD')
    .replace(/[\u2010-\u2015\u2212\uFE58\uFE63\uFF0D]/g, '-')
    .toLowerCase();
}
```

지원 하이픈 문자 (10종): U+2010~U+2015, U+2212, U+FE58, U+FE63, U+FF0D

**적용 범위**: 단일 검색어, AND/OR 복수 검색어, IES 인터페이스 필터링 모두 적용

# Release Notes - v5.8.4

> **Date**: 2026-04-10
> **Branch**: `main`
> **Type**: Bug Fix

## Overview

`extractSection` 함수가 빈 `service` 블록에서 멈춰 일부 Nokia config 파일의 서비스를 전혀 파싱하지 못하는 버그 수정.

## 증상

- `SK-Net_Gyowon_7750SR-a8_MPLS_2.txt` 로드 후 Services 목록에 서비스가 표시되지 않음
- `1640`으로 검색 시 PangyoITC2F에서만 결과가 나오고 Gyowon은 누락
- Epipe 1640 다이어그램에 PangyoITC2F 장비 1개만 표시 (Gyowon 미포함)

## 근본 원인

Nokia 7750SR-a8 config 파일은 `service` 키워드가 **두 군데** 등장:

```
Line 495:  service          ← Oper-Groups 컨텍스트 (빈 블록)
           exit
...
Line 2469: service          ← 실제 Service Configuration (SDP, epipe 등)
               sdp 13000 ...
               epipe 1640 ...
```

`extractSection(configText, 'service')`가 **첫 번째 빈 블록**에서 일치 후 즉시 `exit`를 만나 종료.
실제 서비스 정보가 있는 두 번째 블록에 도달하지 못했음.

## 수정 내용

**`src/utils/v3/parserV3.ts`** — `extractSection` 함수

빈 섹션 감지 후 계속 탐색하는 로직 추가:

```typescript
if (line.trim() === 'exit' && currentIndent === sectionIndent) {
    sectionLines.push(line);
    // 빈 섹션 여부 확인 (start + blanks + exit만 있는 경우)
    const hasContent = sectionLines.some(l => {
        const t = l.trim();
        return t && t !== sectionName && t !== 'exit' && !t.startsWith('#') && !t.startsWith('echo');
    });
    if (!hasContent) {
        // 빈 섹션 — 초기화 후 다음 섹션 계속 탐색
        sectionLines = [];
        inSection = false;
        sectionIndent = -1;
        continue;
    }
    break;
}
```

## 영향 범위

- `extractSection`은 `bgp`, `ospf`, `service` 섹션 파싱에 공통 사용
- 동일한 섹션 이름이 여러 번 등장하는 다른 파일도 이번 수정으로 올바르게 처리됨
- 기존 정상 파싱 동작은 변경 없음 (빈 섹션이 없는 파일에 영향 없음)

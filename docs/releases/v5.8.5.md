# Release Notes - v5.8.5

> **Date**: 2026-04-10
> **Branch**: `main`
> **Type**: UX Enhancement

## Overview

"파일에서도 발견됨" 섹션에서 이미 로드된 파일을 제외하고 미로드 파일만 표시.
로드된 파일은 일반 서비스 목록에 이미 표시되므로 중복 안내가 불필요했음.

## 변경 내용

**`src/components/v3/ServiceListV3.tsx`**

- "파일에서도 발견됨" 섹션 표시 조건 변경
  - 기존: `fileSearchLoading || fileSearchResults.length > 0`
  - 변경: `fileSearchResults.some(r => 미로드 파일 존재)` — 미로드 파일이 1개 이상일 때만 표시
- `fileSearchResults` 렌더링 시 로드된 파일 필터링 (`filter(result => !isLoadedInApp)`)
- "로드됨" 뱃지 제거 (이미 로드된 파일은 섹션 자체에서 제외되므로 불필요)
- 로드 버튼의 `!isLoadedInApp` 조건 제거 (필터링 후 남은 항목은 모두 미로드)

## 동작 변화

| 상황 | 기존 | 변경 후 |
|------|------|---------|
| 로드된 파일만 매칭 | "파일에서도 발견됨" 섹션에 "로드됨" 뱃지와 함께 표시 | 섹션 미표시 |
| 미로드 파일도 매칭 | 로드됨/미로드 혼합 표시 | 미로드 파일만 표시 |
| 미로드 파일만 매칭 | 동일 | 동일 |

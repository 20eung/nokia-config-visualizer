## 변경 내용

### Performance

**React 성능 최적화 (Vercel React Best Practices)**

#### 1. 이벤트 리스너 메모리 누수 수정 (`V3Page.tsx`)

- **증상**: 패널 리사이즈 후 마우스를 빠르게 이동하면 `mousemove` 이벤트 리스너가 해제되지 않고 누적
- **원인**: `resize`, `stopResizing`, `startResizing` 함수가 `useCallback` 없이 정의되어 렌더마다 새 참조 생성 → `removeEventListener` 호출 시 다른 참조 전달로 실제 해제 실패
- **수정**: 세 함수 모두 `useCallback` 적용, `useEffect` 의존성 배열에 `resize`, `stopResizing` 명시
- **추가**: `resize` 내부의 불필요한 `isResizing` 중복 체크 제거 (`useEffect` 가 이미 조건 처리)

#### 2. WebSocket Stale Closure 수정 (`useConfigWebSocket.ts`)

- **증상**: 파일 활성화/비활성화 후 서버에서 파일이 변경되어도 변경 감지가 오동작 가능
- **원인**: `connect()` 내부 `ws.onmessage`가 초기 `handleMessage` 클로저를 캡처. 이후 `activeFiles` 변경으로 `handleMessage`가 교체되어도 `ws.onmessage`는 구버전 참조 유지 (Stale Closure)
- **수정**: `handleMessageRef = useRef(handleMessage)` 추가, `useEffect`로 최신 핸들러 동기화, `ws.onmessage`가 `handleMessageRef.current(message)` 호출하도록 변경

#### 3. 정규식 매 호출마다 재생성 수정 + 중복 함수 공유 유틸 추출

- **증상**: 검색 입력 시마다 `RegExp` 객체가 반복 컴파일
- **원인**: `normalizeSearchString` 함수 내부 정규식 리터럴이 호출마다 새 `RegExp` 생성. 동일 함수가 `Dashboard.tsx`, `ServiceListV3.tsx` 두 파일에 중복 정의
- **수정**: `src/utils/stringUtils.ts` 신규 생성 — `RE_UNICODE_HYPHEN` 정규식을 모듈 레벨 상수로 호이스팅, `normalizeSearchString` 공유 export. 두 파일의 인라인 정의(22줄) 제거 후 공유 유틸 import로 교체
- **참고**: `parserV3.ts`의 기존 `RE_*` 상수 패턴과 일관성 유지

#### 4. `handleCopyImagePNG` useCallback 적용 (`ServiceDiagram.tsx`)

- **증상**: Copy PNG 버튼 클릭마다 ~150줄 규모의 함수 객체 재생성
- **수정**: `handleCopyImagePNG`에 `useCallback` 적용, 의존성 배열 `[diagram, service.serviceId, hostname]` 명시

#### 5. `statCards` useMemo 적용 (`Dashboard.tsx`)

- **증상**: `totalStats`가 변경되지 않아도 매 렌더링마다 새 배열 객체 생성
- **수정**: `statCards` 배열을 `useMemo(() => [...], [totalStats])`로 감싸 참조 안정화

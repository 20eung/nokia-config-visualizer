# Release Notes - v5.7.1

> **Date**: 2026-04-03
> **Branch**: `main`
> **Type**: Bug Fix

## Summary

SK-Net Portal에서 Nokia Visualizer iframe 로드 시 Authentik 인증 흐름이 차단되는 문제를 수정합니다.

## Bug Fix

### 포털 iframe에서 Nokia Visualizer 즉시 접속 불가 문제

**증상**: SK-Net Portal에서 Nokia Visualizer 카드 클릭 시 iframe이 blank 상태로 접속이 안 되는 현상.
Grafana, Gitea, IP Manager 등 다른 서비스는 정상 접속.

**원인**:

v5.7.0에서 추가한 Authentik Forward Auth가 iframe 컨텍스트에서 동작 불가.

```
포털 iframe → ncv.hub.sk-net.com 로드
→ nginx auth_request 실패 (미인증)
→ 302 리다이렉트 → /outpost.goauthentik.io/start
→ Authentik이 auth.hub.sk-net.com 로그인 페이지로 리다이렉트
→ auth.hub.sk-net.com: X-Frame-Options 차단
→ iframe blank / 접속 불가
```

**수정 내용**:

**1. `nginx.conf` — iframe 요청 감지 시 TOP 프레임 리다이렉트**

`Sec-Fetch-Dest: iframe` 헤더로 iframe 컨텍스트를 감지.
인증 실패 시 iframe 내부에서 리다이렉트하는 대신, TOP 프레임(포털 전체 페이지)을 `?_pr=1` 파라미터와 함께 Nokia Visualizer로 이동시킴.

```nginx
map $http_sec_fetch_dest $ncv_iframe_request {
    "iframe"    1;
    default     0;
}

location @goauthentik_proxy_signin {
    if ($ncv_iframe_request) {
        return 200 '<html>...<script>window.top.location.href="https://ncv.hub.sk-net.com/?_pr=1";</script>...</html>';
    }
    return 302 /outpost.goauthentik.io/start?rd=$request_uri;
}
```

**2. `src/main.tsx` — `?_pr=1` 감지 시 포털 자동 복귀**

Nokia Visualizer가 TOP 프레임에서 인증 완료 후 `?_pr=1` 파라미터를 감지하면 포털로 자동 복귀.
React 렌더링 이전에 동기 처리하여 화면 깜빡임 없음.

```typescript
const _params = new URLSearchParams(window.location.search)
if (_params.get('_pr') === '1') {
  window.location.replace('https://portal.hub.sk-net.com/visualizer')
}
```

**수정 후 흐름**:

*최초 인증 시 (1회):*
```
포털 Nokia Visualizer 클릭
→ iframe auth 실패 → TOP 프레임을 ncv.../? _pr=1 로 이동
→ Authentik 인증 (이미 로그인 상태면 자동, 수초 내 완료)
→ main.tsx: ?_pr=1 감지 → portal.../visualizer 자동 복귀
→ iframe 정상 로드
```

*이후 접속 (인증 쿠키 유효):*
```
포털 Nokia Visualizer 클릭 → iframe 즉시 표시
```

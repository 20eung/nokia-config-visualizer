# Release Notes - v5.7.2

> **Date**: 2026-04-10
> **Branch**: `main`
> **Type**: Bug Fix

## 개요

`ServiceDiagram.tsx`의 Mermaid 다이어그램 렌더링 버그 수정.
다이어그램 코드는 정상 생성되지만 화면에 표시되지 않고 에러박스도 나타나지 않는 현상 해결.

## 버그 수정

### Mermaid 렌더링 Silent Failure 수정

**문제**: `await import('mermaid')`가 `try-catch` 블록 **바깥**에 위치하여, 모듈 로드 실패 시 unhandled promise rejection이 발생. 결과적으로 빈 화면만 표시되고 에러박스도 표시되지 않음.

**재현 조건**: v5.7.0에서 추가된 Authentik Forward Auth가 `.js` 청크 파일에도 적용되어, 인증 세션 만료 시 mermaid 청크 요청이 HTML 페이지를 반환 → 모듈 파싱 실패 → silently 빈 화면.

**수정 내용** (`src/components/v3/ServiceDiagram.tsx`):
- `await import('mermaid')` 및 `mermaid.initialize()` 호출을 `try-catch` 블록 안으로 이동
- 모든 렌더링 오류가 에러박스로 표시됨 (이전: silent fail)
- `isMounted` 플래그 추가 → 컴포넌트 언마운트 후 비동기 완료 시 DOM 업데이트 방지
- `useEffect` 의존성 배열에 `hostname` 추가 (stale closure 수정)

```typescript
// Before (버그): import가 try-catch 바깥 → 실패 시 silent fail
const renderDiagram = async () => {
    const { default: mermaid } = await import('mermaid');  // ← try-catch 바깥
    try { ... } catch (error) { /* import 오류를 잡지 못함 */ }
};

// After (수정): import도 try-catch 안으로 → 실패 시 에러박스 표시
const renderDiagram = async () => {
    try {
        const { default: mermaid } = await import('mermaid');  // ← try-catch 안
        ...
    } catch (error) { /* 모든 오류 처리 */ }
};
```

# Release Notes - v4.7.4

> **Date**: 2026-02-21
> **Branch**: `main`
> **Type**: Performance (Minor Release)

## Summary

Vercel React Best Practices 기반 대규모 성능 최적화. 번들 크기 축소 및 런타임 리렌더링을 대폭 개선했습니다.

## Performance Improvements

### 번들 크기 최적화 (Critical)

**lucide-react 배럴 임포트 → 직접 경로 임포트 (14개 파일)**
- `from 'lucide-react'` → `from 'lucide-react/dist/esm/icons/<icon>'`
- Tree-shaking 활성화로 미사용 아이콘 완전 제거

**무거운 라이브러리 동적 임포트**
- `ServiceDiagram.tsx`: mermaid 정적 import → `await import('mermaid')` 동적 import
- `ServiceListV3.tsx`: AIChatPanel, DictionaryEditor → `React.lazy()` + `<Suspense>`

### 리렌더링 최적화 (Medium)

- V3Page 파생 값 10개에 `useMemo` 적용: `allServices`, `selectedServices`, `serviceGroups` 등
- 이벤트 핸들러 `useCallback` 적용: `handleToggleService`, `handleSetSelected`
- Stale closure 해결: `handleServiceSelect`, `handleInterfaceToggle` functional updater 패턴
- `mermaid.initialize()` 최초 1회만 초기화 (모듈 레벨 플래그)

### JavaScript 성능 최적화 (Low-Medium)

- `configs.find()` O(n) → `new Map()` O(1) 조회
- `selectedServiceIds.includes()` O(n) → `Set.has()` O(1)
- `Object.values().filter()` 4회 반복 → 단일 패스 `for...of` 처리
- 원본 배열 변이 방지: `Array.sort()` → `Array.toSorted()` (ES2023)

### 정규식 호이스팅

- `parserV3.ts`: 43개 고정 패턴을 `RE_*` 모듈 상수로 호이스팅
- 대규모 config 파싱 시 20-30% 성능 개선 기대

## Bug Fixes

- TypeScript TS7016 오류: lucide-react 직접 경로 import 타입 선언 추가 (`src/types/lucide-react-paths.d.ts`)

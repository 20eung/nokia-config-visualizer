## 개요

## 변경 내용

### Bug Fix

**IES QoS 엣지 레이블 PNG 렌더링 수정**

- **증상**: IES 서비스 다이어그램에서 Copy PNG 클릭 시, 화면에는 초록색 배지로 표시되던 In-QoS / Out-QoS 레이블이 PNG에서는 스타일 없는 흰 배경 텍스트로 출력됨
- **근본 원인**: 두 가지 QoS 렌더링 경로의 불일치
  - Epipe/VPLS/VPRN: `qosHighlight()` → `<span class='qos-hl' style='...'>` (inline style 포함)
  - IES: `buildQosEdgeLabel()` → `<div class='qos-label'>` (inline style 없음, 외부 CSS만 의존)
  - html2canvas는 SVG `<foreignObject>` 내부에 외부 CSS 미적용 → IES QoS 스타일 완전 소실
  - Canvas 2D 오버레이 코드가 `.qos-hl`만 탐색 → IES의 `.qos-label`은 오버레이에서 누락
- **수정**:
  - `buildQosEdgeLabel()` 반환값에 inline style 추가 (`qosHighlight()`와 동일 패턴)
  - `ServiceDiagram.tsx` 3곳의 CSS 셀렉터를 `.qos-hl, .qos-label`로 확장

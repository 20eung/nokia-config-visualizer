# Release Notes - v5.6.1

> **Date**: 2026-03-19
> **Branch**: `feature/network-type-separation-phase1` -> `main`
> **Type**: Bug Fix + UI Enhancement

## Summary

VPLS 4073 ISP/MPLS 다이어그램 분리 및 networkType 덮어쓰기 버그 수정.
동일 Service ID가 ISP와 MPLS 망에 각각 존재할 때, All 필터에서도 별도 다이어그램으로 표시되며 각 다이어그램에 망 구분 배지가 표시됨.

## Bug Fixes

### 1. networkType 덮어쓰기 버그 (Critical)

**증상**: 서버 재시작 직후에는 ISP/MPLS 분리가 정상이나, 브라우저 접속 후 networkType이 null로 변경됨.

**원인**: 프론트엔드 `useConfigSync` 훅이 `POST /api/ncv/analyze`를 호출하면서 configStore의 AutoParser 데이터(networkType 포함)를 networkType 없는 데이터로 덮어씀.

**수정**:
- `POST /api/ncv/analyze` 엔드포인트에서 기존 configStore의 networkType을 보존
- Incoming 서비스에 preservedNetworkType 적용 (selectionKey suffix 포함)

### 2. ISP/MPLS 다이어그램 합쳐짐

**증상**: All 필터에서 VPLS 4073 ISP와 MPLS가 하나의 다이어그램으로 렌더링됨.

**원인**: `serviceGroups`와 다이어그램 렌더링의 그룹핑 키가 `${serviceType}-${serviceId}` 형태로 networkType을 구분하지 않음.

**수정**:
- `serviceGroups` 그룹핑 키: `vpls-4073` -> `vpls-4073-isp`, `vpls-4073-mpls`
- 다이어그램 렌더링 그룹핑에도 동일 적용

### 3. 프론트엔드 networkType 미전파

**증상**: 프론트엔드 파서가 config 파싱 시 networkType 옵션 없이 호출되어 서비스에 networkType 미설정.

**수정**:
- Backend `GET /api/config/file/:filename`에 `X-Network-Type` 응답 헤더 추가
- 프론트엔드 파일 fetch 시 헤더를 읽어 파서에 `{ networkType }` 옵션 전달

## UI Changes

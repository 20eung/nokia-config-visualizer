# Release Notes - v5.7.0

> **Date**: 2026-04-02
> **Branch**: `main`
> **Type**: Feature (Security)

## Summary

Authentik SSO Forward Auth 연동으로 Nokia Config Visualizer 인증 게이트 구현.
미인증 사용자는 Authentik 로그인 페이지로 자동 리다이렉트되며, 포털과 쿠키 도메인 공유로 SSO 단일 로그인 경험 제공.

## New Features

### 1. Authentik Forward Auth 인증 게이트 (nginx)

**배경**: 서버에 모든 장비 config가 저장되어 있어 비인가 접근 차단 필요.

**구현**: nginx `auth_request` 모듈로 Authentik Embedded Outpost에 인증 위임.

```
미인증 접속 → nginx auth_request → Authentik 401
  → 302 /outpost.goauthentik.io/start?rd=<원래URL>
  → Authentik 로그인
  → 원래 URL 복귀 (세션 쿠키 발급)
```

**적용 경로**: `/`, `/api/*`, `/ws`, 정적 파일 전체

**예외 경로** (인증 없음):
- `/health` — 포털 헬스체크 모니터링용
- `/outpost.goauthentik.io` — Authentik Outpost 통신용 (무한루프 방지)

**변경 파일**: [`nginx.conf`](../../nginx.conf)

### 2. 포털 iframe 임베딩 허용 (CSP frame-ancestors)

**배경**: SK-Net Portal에서 Nokia Visualizer를 iframe으로 임베딩하여 사용 중.

**변경**:
- `X-Frame-Options: SAMEORIGIN` 제거
- `Content-Security-Policy`에 `frame-ancestors 'self' https://portal.hub.sk-net.com` 추가

포털 외 다른 도메인의 iframe 임베딩은 여전히 차단.

### 3. SSO 쿠키 도메인 공유 (hub.sk-net.com)

**배경**: 포털 로그인 후 Nokia Visualizer iframe이 별도 로그인을 요구하는 UX 문제.

**구현**: Authentik Proxy Provider `cookie_domain`을 `hub.sk-net.com`으로 설정.

```
포털(portal.hub.sk-net.com) 로그인
  → 쿠키 Domain=hub.sk-net.com 발급
  → Nokia Visualizer(ncv.hub.sk-net.com) iframe 자동 인증
```

대상 Provider: `portal-proxy`, `nokia-visualizer-proxy`

### 4. 포털 헬스체크 URL 수정

**배경**: nginx에 auth_request 적용 후 포털 헬스체크가 500 반환하여 UNHEALTHY 표시.

**변경**: `services.py` Nokia Visualizer URL을 `http://nokia-visualizer:80/health`로 수정.

**변경 파일**: [`/data/sknet-portal/backend/routers/services.py`](../../../sknet-portal/backend/routers/services.py)

## Security

- 미인증 접근 완전 차단 (모든 경로)
- 포털 도메인에서만 iframe 허용 (CSP frame-ancestors)
- Authentik Embedded Outpost 재사용으로 추가 인프라 없음

# Release Notes - v4.6.1

> **Date**: 2026-02-19
> **Branch**: `main`
> **Type**: Bug Fix (Patch Release)

## Summary

복수 검색어(AND/OR) 사용 시 catch-all 검색이 동작하지 않던 문제 수정. 백엔드 TypeScript 컴파일 오류도 함께 수정.

## Bug Fixes

### 복수 검색어 Catch-all 검색 강화

**증상**: 복수 검색어(AND: `+`, OR: 띄어쓰기)에서 catch-all 검색이 작동 안 함

**수정**: `JSON.stringify(service)`를 searchFields에 추가

- 단일 검색어에서만 작동하던 catch-all이 복수 검색어에서도 동작
- 필드명 자체(예: "portId", "portDescription")도 AND/OR 검색 가능
- 명시적으로 수집하지 못한 필드나 중첩된 객체 내부 값도 검색 가능

### 백엔드 TypeScript 컴파일 오류 수정

`server/src/routes/config.ts`에서 `path.basename(filename as string)` 타입 캐스팅 추가

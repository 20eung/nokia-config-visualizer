# Release Notes - v5.6.2

> **Date**: 2026-03-21
> **Branch**: `main`
> **Type**: Bug Fix (Patch Release)

## Summary

Dashboard 검색창에서 U+2011 Non-Breaking Hyphen이 포함된 hostname 검색이 불가능한 문제 수정.

## Bug Fixes

### Dashboard 검색 Unicode 하이픈 정규화 수정

**증상**: Dashboard 검색창에 `SK‑Net_Gyowon_7705SAR‑8_MPLS` 입력 시 검색 결과 없음

**원인**: `Dashboard.tsx`의 검색 필터가 `.toLowerCase()`만 사용하여 U+2011 Non-Breaking Hyphen을 일반 하이픈(U+002D)으로 변환하지 않아 hostname과 매칭 실패.

```typescript
// Before (버그)
const q = searchQuery.toLowerCase();
g.hostnames.some(h => h.toLowerCase().includes(q))

// After (수정)
const q = normalizeSearchString(searchQuery);
g.hostnames.some(h => normalizeSearchString(h).includes(q))
```

**수정**: `normalizeSearchString()` 함수를 `Dashboard.tsx`에 추가하여 `ServiceListV3.tsx`와 동일한 정규화 로직 적용.

```typescript
function normalizeSearchString(str: string): string {
  return str
    .normalize('NFKD')
    .replace(/[\u2010-\u2015\u2212\uFE58\uFE63\uFF0D]/g, '-')
    .toLowerCase();
}
```

**적용 범위**: `siteName` 및 `hostnames` 필터 모두 정규화 적용

## Background

v5.5.2에서 `ServiceListV3.tsx`의 Services 탭 검색에 Unicode 정규화를 적용했으나, `Dashboard.tsx`의 사이트 검색에는 동일 로직이 누락되어 있었음.

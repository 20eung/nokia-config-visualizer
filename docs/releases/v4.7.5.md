# Release Notes - v4.7.5

> **Date**: 2026-02-21
> **Branch**: `main`
> **Type**: Performance (Patch Release)

## Summary

v4.7.4에 이어 Vercel React Best Practices 기반 추가 성능 최적화. 라우트 레벨 코드 스플리팅 및 순수 컴포넌트 메모이제이션 적용.

## Performance Improvements

### 번들 크기 최적화 (Critical)

**라우트 레벨 코드 스플리팅**
- `App.tsx`: V3Page, V1Page를 `React.lazy()` + `<Suspense>` 로 변환
- 초기 번들 ~420KB 추가 절감 (라우트별 지연 로딩)

### 리렌더링 최적화 (Medium)

**순수 컴포넌트에 `React.memo()` 적용**:
- `ServiceDiagram`: mermaid 재렌더링 방지 (검색/필터 상태 변경 시)
- `AliasBadge`: 리스트 내 다수 인스턴스 불필요한 재렌더 방지
- `ConfigFileList`: 파일 목록/activeFiles 불변 시 재렌더 방지

**useLatest 패턴 (Stale closure 해결)**
- `V3Page.tsx`: `config-file-changed` 이벤트 핸들러의 `activeFiles` stale closure 해결
- `useEffect` 의존성 배열 `[activeFiles]` → `[]` (마운트 1회만 등록)

### JavaScript 성능 (Low-Medium)

- `Array.sort()` → `Array.toSorted()` 추가 적용 (TopologyEngine.ts, DictionaryEditor.tsx)

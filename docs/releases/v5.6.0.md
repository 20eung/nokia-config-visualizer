# Release Notes - v5.6.0

> **Date**: 2026-03-19
> **Branch**: `main`
> **Type**: Feature (Minor Release)

## Summary

Network Type Separation Phase 1 구현. ISP/MPLS/Cloud 망을 파일 경로 기반으로 자동 구분하여 동일 Service ID가 다른 망에 존재할 때 독립된 토폴로지 뷰를 생성합니다.

## New Features

### Network Type Separation

**Network Type 분류** (`server/src/utils/networkTypeExtractor.ts`):
- 파일 경로 기반 networkType 자동 추출
  - `/configs/isp/` → `isp`
  - `/configs/mpls/` → `mpls`
  - `/configs/cloud/` → `cloud`
- Hybrid 전략: 경로 → 파일명 → Config 내용 추론

**selectionKey 생성 로직 개선**:
- 이전: `${serviceType}-${serviceId}` (예: `vpls-4073`)
- 신규: `${serviceType}-${serviceId}-${networkType}` (예: `vpls-4073-isp`, `vpls-4073-mpls`)
- Fallback: networkType이 `unknown`이면 기존 포맷 사용 (하위 호환)

**Service ID 충돌 해결 사례**:
- VPLS 4073 ISP (`SVC_SKSH_Internet_2G`) vs VPLS 4073 MPLS (`Gyowon_Access-Control`)
- 결과: 독립된 토폴로지 뷰 생성 (ISP 망, MPLS 망 분리)

### Frontend UI

- Network Type 필터 버튼 (ISP/MPLS/CLOUD) 추가
- Network Type 배지: Cyan=ISP, Purple=MPLS, Slate=Cloud
- HA 감지 로직에 networkType 반영

## Backend Changes

**타입 정의 추가**:
- `src/types/services.ts`: `NetworkType` 타입 (`'isp' | 'mpls' | 'cloud' | 'unknown'`)
- `server/src/types.ts`: `ServiceSummary.networkType` 필드
- `server/src/services/configStore.ts`: `StoredConfig.networkType`

**Parser 통합**:
- `src/utils/v3/parserV3.ts`: `ParseOptions` 인터페이스 추가
- `server/src/services/nokiaParserCore.ts`: ParseOptions 파라미터 지원
- `server/src/services/autoParser.ts`: extractNetworkType 적용

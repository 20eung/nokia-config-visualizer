# Release Notes - v5.8.3

> **Date**: 2026-04-10
> **Branch**: `main`
> **Type**: Feature Enhancement

## Overview

watchFolder 내 **전체 파일 텍스트 검색** 기능 추가 (search-global-config).
앱에 로드되지 않은 파일에서도 서비스를 발견하고, 클릭 한 번으로 즉시 로드할 수 있습니다.

## 배경

`1640`으로 검색 시 앱은 1개 결과만 반환했지만, `/data/configs/mpls`에서 grep 실행 시 2개 파일에서 발견:

```
SK-Net_Gyowon_7750SR-a8_MPLS_2.txt    ← 앱에 미로드 → 검색 누락
SKNet_PangyoITC2F_7750SR_MPLS_2.txt   ← 앱에 로드됨
```

**근본 원인**: 검색 대상이 `props.services` (로드된 configs)로 한정되어 미로드 파일의 서비스가 누락됨.

## Changes

### Backend

**`server/src/services/configStore.ts`**
- `getLoadedFilenames(): Set<string>` 메서드 추가 — isLoaded 판정용

**`server/src/routes/config.ts`**
- `GET /api/config/search-files?q={query}` 엔드포인트 추가
  - watchFolder 내 전체 `.txt` 파일 대상 텍스트 검색 (라인별)
  - 검색어 3자 미만 시 `400` 반환
  - 10MB 이상 파일 자동 건너뜀
  - snippet 최대 3개 (120자 제한)
  - 미로드 파일 우선 정렬, 같은 그룹 내 매칭 수 내림차순

### Frontend

**`src/services/configApi.ts`** _(신규 생성)_
- `FileSearchResult` interface 정의
- `searchConfigFiles(query, signal?)` 함수 — AbortSignal 지원

**`src/components/v3/ServiceListV3.tsx`**
- `onLoadFile?: (filename: string) => void` prop 추가
- 파일 검색 state 3개: `fileSearchResults`, `fileSearchLoading`, `fileSearchAbortRef`
- debounce 500ms useEffect — 검색어 3자 이상, AI 모드 비활성 시 API 호출
- "파일에서도 발견됨" UI 섹션 추가 (서비스 목록 하단)
  - 파일명, 매칭 수, snippet 미리보기 표시
  - 로드된 파일은 "로드됨" 뱃지 표시
  - 미로드 파일에 `[로드]` 버튼 표시

**`src/pages/V3Page.tsx`**
- `handleLoadFile` useCallback 추가
  - `/api/config/file/{filename}` fetch → parse → 기존 configs에 추가 (중복 hostname은 교체)
- `<ServiceListV3 onLoadFile={handleLoadFile} />` prop 전달

## 사용법

1. watchFolder가 설정된 상태에서 Services 검색창에 3자 이상 입력
2. 0.5초 후 자동으로 파일시스템 전수 검색 실행
3. 미로드 파일이 있으면 목록 하단에 **"파일에서도 발견됨"** 섹션 표시
4. `[로드]` 버튼 클릭 → 해당 파일이 현재 configs에 즉시 추가

## Technical Notes

- AI 모드 활성화 시 파일 검색 미실행 (불필요한 API 호출 방지)
- AbortController로 이전 요청 즉시 취소 (debounce와 연동)
- V3Page의 로드 방식: `handleConfigLoaded` 교체 대신 `setConfigs(prev => [...prev, parsed])` append — 기존 로드된 configs 유지

## Follow-up Fix (v5.8.3)

**문제**: 백엔드 `configStore.isLoaded`와 프론트엔드 `configs` 상태 불일치
- AutoParser가 파일을 인덱싱하면 `configStore`에 등록됨 → `isLoaded: true`
- 하지만 프론트엔드 `configs`에는 미로드일 수 있어 "로드됨" 뱃지 + [로드] 버튼 동시 표시 버그 발생

**수정 내용**:
- `ServiceListV3`: `loadedFilenames?: Set<string>` prop 추가 — 프론트엔드 실제 상태 우선 적용
  ```typescript
  const isLoadedInApp = loadedFilenames?.has(result.filename) ?? result.isLoaded;
  ```
- `V3Page`: `loadedFilenames` useMemo 추가 (activeFiles / uploadedFiles 기반)
- `V3Page`: `handleLoadFile` 개선
  - 기존 선택 서비스와 동일한 `serviceId`의 신규 서비스 자동 선택
  - 로드 완료 후 Services 뷰로 자동 전환 (`setViewMode('services')`)
  - 다이어그램이 즉시 업데이트됨
